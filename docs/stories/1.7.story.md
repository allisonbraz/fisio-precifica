# Story 1.7 — Melhoria das Reservas: Percentual e Impacto no Preço

**Epic:** Correção do Modelo de Precificação
**Status:** Ready
**Prioridade:** MÉDIA
**Agente:** @dev
**Depende de:** Story 1.1

---

## Contexto

Reservas atualmente são apenas valores fixos em R$. O fisioterapeuta não pode dizer "quero reservar 10% da receita". Além disso, a página Reservas não mostra como as reservas impactam o preço sugerido.

## Critérios de Aceite

### AC-1: Modo percentual para reservas
- [ ] Adicionar campo `modo: 'fixo' | 'percentual'` em ReservaEstrategica
- [ ] Se modo = 'percentual', campo `percentual: number` (0-100)
- [ ] Valor mensal calculado: `receita estimada × percentual / 100`
- [ ] Toggle na UI: "Valor fixo" ↔ "% da receita"
- [ ] Ao mudar o preço, reservas percentuais recalculam automaticamente

### AC-2: Reservas recomendadas com auto-cálculo
- [ ] "Férias + 1/3": auto-sugerir `proLabore / 12 * 1.33`
- [ ] "13º equivalente": auto-sugerir `proLabore / 12`
- [ ] Botão "Calcular automaticamente" para cada item
- [ ] Se pró-labore existe nos custos fixos, detectar automaticamente

### AC-3: Impacto visual das reservas no preço
- [ ] Na página Reservas, mostrar card:
  - "Suas reservas adicionam **R$ X,XX** ao custo por sessão"
  - "Seu preço precisa ser pelo menos **R$ Y,YY** para cobrir custos + reservas"
  - Comparativo: "Sem reservas: R$150 | Com reservas: R$168"

### AC-4: Alerta de reservas insuficientes
- [ ] Se total de reservas < 15% da receita: alerta amarelo
  - "Considere aumentar suas reservas. O recomendado é 20-27% da receita."
- [ ] Se total de reservas > 35% da receita: alerta de atenção
  - "Reservas elevadas. Verifique se o preço resultante é competitivo."
