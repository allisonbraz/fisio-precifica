# Story 1.1 — Reformular Fórmula de Precificação

**Epic:** Correção do Modelo de Precificação
**Status:** Ready
**Prioridade:** CRÍTICA
**Agente:** @dev

---

## Contexto

A fórmula atual calcula o preço como `custoTotal × (1 + margem)`, que é um **markup sobre custo**. Isso causa 3 problemas graves:

1. **Reservas ficam de fora do preço** — o usuário cadastra reservas mas o preço sugerido não as cobre
2. **Impostos sobre receita não são considerados** — o preço não embute a carga tributária
3. **A "margem" é na verdade markup** — 25% de "margem" = 20% da receita real, induzindo erro
4. **Custo fixo alocado assume 100% de ocupação** — divide pelo sessoesMeta que pode ser a capacidade total

## Fórmula Atual (INCORRETA)

```typescript
// calculations.ts:49-52
function calcularPrecoMinimo(data): number {
  const custoTotal = calcularCustoTotalPorSessao(data); // fixo + variável, sem reservas
  return custoTotal * (1 + data.margemLucro);            // markup, não margem
}

// calcularCustoTotalMensal: NÃO inclui reservas
// calcularMargemDoPreco: (preco - custo) / custo  → markup, não margem
```

## Fórmula Correta (A IMPLEMENTAR)

```
Preço = (Custo Fixo Alocado + Custo Variável por Sessão) / (1 - Impostos% - Reservas% - Margem%)
```

Onde:
- **Custo Fixo Alocado** = Custos fixos mensais / sessoesMeta
- **Custo Variável por Sessão** = Custos variáveis mensais / sessoesMeta
- **Impostos%** = Alíquota sobre receita (0% a 20%) — novo campo
- **Reservas%** = Total reservas mensais / Receita estimada — ou calculado iterativamente
- **Margem%** = Margem de lucro desejada como % da receita (não do custo)

### Abordagem Simplificada (Recomendada para V1)

Como Reservas% depende da receita (circular), usar abordagem de **custo total expandido**:

```typescript
function calcularPrecoMinimo(data: DadosPrecificacao): number {
  const custoFixoMensal = calcularTotalCustosFixos(data.custosFixos);
  const custoVarMensal = calcularTotalCustosVariaveis(data.custosVariaveis);
  const reservasMensal = calcularTotalReservas(data.reservasEstrategicas);

  // Custo total mensal EXPANDIDO (inclui reservas como custo)
  const custoTotalMensal = custoFixoMensal + custoVarMensal + reservasMensal;

  if (data.sessoesMeta === 0) return 0;
  const custoPorSessao = custoTotalMensal / data.sessoesMeta;

  // Margem agora é sobre a RECEITA: preço = custo / (1 - margem% - imposto%)
  const divisor = 1 - data.margemLucro - data.impostoPercentual;
  if (divisor <= 0) return custoPorSessao * 10; // cap de segurança

  return custoPorSessao / divisor;
}
```

## Notas Técnicas

### Margem vs Markup — Conversão
- Markup 25% → Margem 20%: `0.25 / (1 + 0.25) = 0.20`
- Margem 15% → Markup 17.6%: `0.15 / (1 - 0.15) = 0.176`

### Por que Margem sobre Receita?
1. Benchmarks de saúde usam margem (não markup)
2. O método SEBRAE usa margem internamente
3. Margem tem teto natural de 100%
4. Mais intuitivo: "15% do que eu recebo é lucro"

### Reservas como Custo — Justificativa Contábil
Provisões são despesas reconhecidas no período (CPC 25). Para autônomos, férias/13º/emergência são obrigações futuras certas. Incluí-las no preço garante solvência real.
