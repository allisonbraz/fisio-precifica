# Story 1.5 — UX Educacional: Explicações e Orientação de Workflow

**Epic:** Correção do Modelo de Precificação
**Status:** Ready
**Prioridade:** ALTA
**Agente:** @dev

---

## Contexto

O fisioterapeuta não é contador nem administrador. O app precisa educar sobre os conceitos enquanto o usuário preenche. Problemas identificados:

1. **Sem orientação de por onde começar** — Home mostra zeros sem nudge
2. **Margem não explicada** — usuário não sabe se é sobre custo ou receita
3. **Depreciação vs Financiamento confuso** — pode duplicar custos
4. **Ponto de Equilíbrio sem contexto** — número solto, sem ação
5. **Benchmarks ausentes** — "60% ocupação" é bom ou ruim?

## Critérios de Aceite

### AC-1: Onboarding para usuário novo (Home com dados vazios)
- [ ] Se custos fixos estão todos zerados, mostrar banner:
  - "Comece cadastrando seus custos para calcular seu preço ideal"
  - Botão "Começar pelos Custos →"
- [ ] Stepper visual no topo: ① Custos → ② Precificação → ③ Serviços → ④ Simulação
- [ ] Steps completados ficam verdes (baseado em dados preenchidos)

### AC-2: Explicação de Margem na página Precificação
- [ ] Info box abaixo do slider de margem:
  - "**Margem de 15%** significa que de cada R$100 que você recebe, R$15 é lucro líquido."
  - Atualizar dinamicamente com o valor do slider
- [ ] Bandas de referência:
  - < 10%: "Arriscada — pouca folga para imprevistos"
  - 10-20%: "Saudável — margem equilibrada"
  - 20-30%: "Confortável — boa reserva de segurança"
  - > 30%: "Excelente — verifique se o preço é competitivo"

### AC-3: Guia de Depreciação vs Financiamento na página Custos
- [ ] Na aba Depreciação, info box permanente:
  - "**Financiamento ativo?** Se você está pagando parcelas de um equipamento, coloque em Custos Operacionais (não aqui)."
  - "**Depreciação** é para equipamentos já pagos — reserve para repor quando desgastar."
- [ ] Se `temParcelaAtiva = true`, mostrar alerta:
  - "Este item tem parcela ativa. O valor da parcela já está em Custos Operacionais. Depreciação será considerada após quitação."

### AC-4: Ponto de Equilíbrio contextualizado
- [ ] Onde o PE aparecer, adicionar contexto:
  - "Você precisa de **X sessões/mês** para cobrir todos os custos."
  - "Sua capacidade é **Y sessões/mês** (Z% de ocupação necessária)."
  - Se PE > 80% capacidade: alerta "Margem de segurança baixa"
  - Se PE < 50% capacidade: "Boa margem de segurança"

### AC-5: Benchmarks de mercado
- [ ] Na página Indicadores, adicionar faixas de referência:
  - Taxa de Ocupação: "Clínicas saudáveis: 60-75%"
  - Margem: "Serviços de saúde: 15-25%"
  - Break-even: "Ideal: abaixo de 50% da capacidade"
- [ ] Mostrar como barra de referência no fundo dos cards
- [ ] Disclaimer: "Referências baseadas em dados do setor de fisioterapia"

### AC-6: Tooltips educacionais em métricas-chave
- [ ] Cada StatCard com ícone (?) que mostra tooltip:
  - Custo por Sessão: "Total dos seus custos mensais dividido pela meta de sessões"
  - Taxa de Ocupação: "Meta de sessões ÷ capacidade máxima. Ex: 60 de 100 = 60%"
  - Valor/Hora: "Preço da sessão convertido para 1 hora de trabalho"
  - Pró-labore: "Sua remuneração pelo trabalho — é custo, não lucro"
