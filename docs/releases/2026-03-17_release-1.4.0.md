## Causa Raiz — Vazamento de Dados (P0)

### Problema
Os endpoints `pricing.load` e `pricing.save` em `server/routers.ts` usavam `publicProcedure` (sem autenticação) e recebiam o email como parâmetro do frontend. Qualquer pessoa podia requisitar dados de qualquer email.

Adicionalmente, o `localStorage` do browser não era isolado por usuário — ao trocar de conta, os dados da sessão anterior permaneciam.

### Correção

| Camada | Antes | Depois |
|--------|-------|--------|
| **API** | `publicProcedure` + `input.email` | `protectedProcedure` + `ctx.user.email` |
| **Client** | localStorage compartilhado | Reset automático ao detectar troca de conta |
| **Input** | `{ email, data }` / `{ email }` | `{ data }` / sem input |
